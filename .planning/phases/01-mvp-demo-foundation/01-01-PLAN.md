# Plan 01-01: Build ProfitPilot MVP Demo Foundation

<objective>
Build the deployable ProfitPilot MVP demo foundation from `docs/plans/2026-06-15-profitpilot-mvp-implementation.md` using strict TDD: write failing tests first, verify failure, implement minimal passing code, then verify green.
</objective>

## Source Plan

Follow the task sequence in:

- `docs/plans/2026-06-15-profitpilot-mvp-implementation.md`

## Task 1: Initialize Application Skeleton

Write and fail the smoke test, then create the minimal Next.js, TypeScript, Tailwind, and Vitest application shell.

## Task 2: Add Domain Types And Mock Data

Write and fail the mock-data linkage test, then add typed records and realistic linked sample data.

## Task 3: Build Profit Calculation Engine

Write and fail the profit approval test, then implement deterministic profit and margin analysis.

## Task 4: Build Risk And Automation Engines

Write and fail risk and automation tests, then implement the rule-based engines.

## Task 5: Create Dashboard Data API

Write and fail the dashboard summary test, then aggregate mock data and expose `GET /api/analytics/dashboard`.

## Task 6: Build Command Center UI

Extend and fail the UI smoke test, then build the main dashboard components.

## Task 7: Add Product Packet And Listing Studio Views

Write and fail the product packet test, then add the product workspace route and components.

## Task 8: Add Database Schema And Render Deployment

Write and fail the Render config test, then add Prisma schema, Render Blueprint, and `.env.example`.

## Task 9: Verify Build, Tests, And Local App

Run `npm test`, `npm run build`, and start the local app. Fix only defects exposed by verification.

## Task 10: Deploy To Render

Deploy only after source is pushed or otherwise available to Render. Use the Render API key only as a credential, never in committed files.
